# Home Assistant Setup — Bootstrap Notes

Home Assistant runs on a dedicated VM (`hass`, `192.168.1.89:8123`), not in
Kubernetes. The manifests in this directory expose it through Traefik at
`https://hass.example.com`.

## EndpointSlice must be applied manually on first deploy

The Service in `ingress.yaml` has no selector — it points at an external IP
(the HA VM). For that pattern, Kubernetes requires us to provide the
`EndpointSlice` ourselves rather than auto-generating one from pod labels.

ArgoCD's default `resource.exclusions` list drops both `Endpoints` and
`EndpointSlice` resources cluster-wide (they're noisy when auto-generated by
the Service controller). That default is correct for the 95% case, but it
means our manual EndpointSlice gets silently skipped during ArgoCD sync.

**Symptom:** the Service exists but `kubectl describe service home-assistant`
shows `Endpoints: <none>`, and Traefik returns `503 no available server`.

**Fix:** apply `ingress.yaml` once manually after first deploy.

```bash
kubectl apply -f k8s/home-assistant/ingress.yaml
```

Manual EndpointSlices aren't owned by the Service controller, so they persist
indefinitely. Repeat this only if someone deletes the EndpointSlice or if the
backend IP changes (edit the file first).

## HA-side configuration

The companion repo [my-home](https://git.example.com/my/my-home)
is auto-pulled to `/config` on the VM via a Forgejo webhook. The relevant
HTTP ingress block lives there:

```yaml
homeassistant:
  external_url: "https://hass.example.com"
  internal_url: "http://192.168.1.89:8123"

http:
  use_x_forwarded_for: true
  trusted_proxies:
    - 192.168.1.0/24    # k0s worker nodes (SNAT'd egress)
    - 10.244.0.0/16   # k0s pod CIDR
```

Without `trusted_proxies`, HA returns `400 Bad Request` to anything coming
through Traefik. Changes to the `http:` block require a full HA restart
(Settings → System → Restart) — they aren't picked up by Developer Tools →
YAML reload.

## DNS

`hass.example.com` must resolve to Traefik (`192.168.1.87`), not
directly to the VM. This is the default behavior of the Pi-hole wildcard
(`*.example.com → traefik_ip`) — just don't add `hass` to
`dns_direct_hosts` in `ansible/inventory/group_vars/all.yml`.

The VM IP (`192.168.1.89`) is still reachable directly from the LAN for
break-glass admin work.
